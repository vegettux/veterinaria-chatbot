# Reglas SDD del proyecto

Este proyecto sigue metodología Spec Driven Development (SDD):

- Todo desarrollo parte de un ticket en Jira
- Cada funcionalidad está vinculada a un ticket
- El repositorio GitHub refleja la implementación
- Jira refleja planificación y estado

Reglas:
- No se implementa sin ticket
- No se cierra ticket sin validación funcional
- Jira y GitHub están alineados
